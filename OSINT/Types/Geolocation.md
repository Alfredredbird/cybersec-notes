# Pretext

Geolocation is form of OSINT that is about finding and pinpointing locations from a image or form of media. I am only going to cover the basics as it can be very complex.

# Methodology

When Geolocating, it's important to gather as much as we can to then compare it with active map data. We want to be able to find identifiable objects that we can use to find secure a possible location. We will use this image today.
![[osintimage1.jpg]]
> Image from https://www.geograph.org.uk/photo/7388823
# Extracting

I often refer to the process of collecting important objects as `extracting`.
When we are extracting, we want to look for objects that will be of use to us. 
We will first start with gathering buildings, signs, landmarks and vehicles.
![[osintimage2.jpg]]
In our subject image, I was able to identify a Church `(A)`, a building `(b)`, white license plate `(c)` and a yellow license plate `(d)`.
From these findings, we can assume it's in the United Kingdom since the cars have a white plate on the front and yellow on the back.
# Reverse Image Search

When it comes to reverse image searching, we want to narrow down our points to see if we can reverse the images back to the possible source or maybe find the landmark. We will be using points `A` and `B`. Some good search images are [Yandex](https://yandex.com/) and [Google Images](https://images.google.com/). After looking at some of the images on Yandex, we can see that the building with the marker `B` is the `King Henry Terrace` Building.
# The Triad

`The Triad` is what I like to call our final point markup. To make the triad, you simply need to draw a triangle from each point that we used as a marker. Does it have to be a triangle, no. There is no need to do this but it looks cool.
